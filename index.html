This is Gravity Sort

Gravity sort is a natural sorting algorithm developed by Joshua Arulananandham, Cristian Calude, and Micahel Dinneen in 2002.

It was designed to imitate beads falling under the influence of gravity. It can only be used to sort lists of positive integers.

Implementation in Java


                            public static int[] beadsort(int[] array) {
                                int max = array[0];
                                for (int i = 1; i < array.length; i++) {
                                    if (max < array[i]) {
                                        max = array[i];
                                    }
                                }
                                int[] sorted = new int[array.length];
                                int[] counter = new int[max];
                                for (int value : array) {
                                    for (int j = 0; value > 0; value--, j++) {
                                        sorted[counter[j]++]++;
                                    }
                                }
                                return sorted;
                            }

Space and Time Complexity

The space complexity of the above implementation is θ(n + m), where n is number of elements to be sorted and m is the largest number from the array.

The time complexity is θ(n) with the right hardware.

Otherwise, the time complexity is O(S), where S is the sum of all the numbers.

Use Cases

  • With the right hardware, you can sort positive numbers in linear time.
  • Not very useful for industry because it can only sort positive numbers.
  • If one of the numbers is significantly larger than the number of elements in the array, the space complexity is O(m). The run time would still be O(S), where S is the sum of all numbers, but S would be an incredibly large number.
